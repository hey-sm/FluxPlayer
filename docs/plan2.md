## 目标与边界

本轮在现有 TopBar、悬浮侧栏、搜索和 Three.js 舞台修正计划上继续收敛，新增以下明确目标：

1. 删除界面主题预设配置，应用只保留并固定使用“经典（classic-gold）”主题。
2. 滚动、拖动侧栏、拖动歌词或操作滑块时，不得选中页面文字或出现图片拖拽残影。
3. 将沉浸式画面按职责拆成“壁纸层”和“歌词层”：歌词层始终保留 3D 与滚轮/拖动交互；壁纸层只有在 Three.js 3D 视觉模式下才响应滚轮。
4. 升级 Three.js 并新增多套 3D 音乐背景，在不破坏现有预设编号和经典视觉的前提下逐项迁移。
5. 引入 `react-glass-ui` 与 `gsap`：玻璃组件逐步替换重复的手写玻璃容器，GSAP 替换适合交由时间轴管理的手写 UI/歌词动画。
6. 歌词支持逐字时间轴与多种可切换动效；播放音质控件改为外部直接显示当前具体音质、点击后向上展开切换。

Three.js 从当前 `0.128.0` 升级到实施时锁定并验证的目标版本，`three` 与 `@types/three` 必须保持同版；当前调研版本为 `0.185.1`，最终版本写入 lockfile 后不使用浮动范围。升级不允许同时重写全部 shader：先建立基线截图与兼容层，再按编译、运行、视觉回归分阶段迁移。现有六个视觉预设数字 ID 不重排，新背景只追加新 ID。双层仍是职责与事件路由分层，优先复用一个 `VisualStage`、一个 Three.js scene 和一个 WebGL renderer，避免创建第二套 GPU 上下文。

`react-glass-ui@1.2.2` 当前 peer 范围支持 React 19，`gsap@3.15.0` 可直接用于 React/DOM/SVG/Three 对象动画；实施时使用精确版本并提交 lockfile。第三方玻璃组件先通过本项目适配层接入，不直接散落业务页面，也不一次性替换经典主题的核心 SVG/filter 资产。

## 已确认的现状

- 设置目前包含六套**界面主题预设**及一组可调视觉参数；这与 Three.js 的六个**音乐视觉预设**是两套系统。
- Three.js 音乐视觉预设（丝绸、滚筒、轨道、虚空、唱片、星河）继续保留；本次删除的仅是界面主题预设选择，不得删除或重排视觉预设 ABI。
- 自定义图片、视频或 Wallpaper Engine 背景启用后，`StageCanvas` 会通过 `backgroundEnabled=false` 暂停背景对象，但当前也连带停止了相机更新与歌词组旋转。
- 当前画布使用全局 capture 鼠标事件；滚轮只粗略排除部分表单控件，容易抢走搜索结果、歌单、设置等原生滚动。
- 当前拖动封面使用 `mousedown/mousemove`，未形成统一的拖动状态和文本选择保护。
- 搜索感应区仍存在被 TopBar 层级覆盖的风险；左右音乐侧栏仍需按非模态悬浮侧栏方案收敛。

## 一、界面主题只保留经典

- 删除设置中的“界面主题预设”卡片、下拉框、切换入口以及主题自定义/恢复预设交互，不再让用户选择默认暗色、浓雾玻璃、清透玻璃、液态玻璃和皓白。
- `classic-gold` 成为唯一内置主题和启动默认值；经典主题参数保留为代码常量，不再作为一组用户可选预设配置暴露。
- 删除五套非经典主题定义以及只服务于多预设切换的列表、ID 联合类型和 store action；保留应用实际消费的经典主题 token/DOM 变量应用能力。
- 对旧版本地持久化数据做一次兼容迁移：无论旧值是哪个主题或自定义快照，升级后统一落到经典主题；忽略无法识别的旧字段，启动不得报错。
- 不修改 `src/renderer/src/theme/classic/` 的核心经典玻璃资产及其逐值效果。
- 设置弹窗的标题/说明改为与现状一致，不再暗示存在多套外观主题；音乐视觉预设仍放在“音乐视觉”区域。

## 二、TopBar、悬浮侧栏与搜索

- TopBar 完全透明：删除背景填充、边框、阴影和 backdrop blur，只保留 48px 拖拽层、Logo、`FLUXP`、设置及窗口控制按钮；按钮保留 hover/focus 反馈。
- 删除 TopBar 的“音乐库”和“当前歌单”按钮及对应 `ActiveSheet` 参数；保留设置、最小化、全屏和关闭。
- 左右栏不再使用模态 Radix Sheet，恢复非模态移入呼出：
  - 左右边缘使用居中半高感应区，面板从 48px TopBar 下方开始。
  - 移入感应区或面板时展开，离开后延迟 2 秒关闭；重新移入取消关闭计时。
  - 左栏启动时可见，开始播放后自动收起；选择歌单、喜欢或最近播放时展开右栏。
  - 左右栏相互独立，不显示遮罩、不抢焦点，也不阻止壁纸、歌词和播放器操作。
  - shadcn `Sheet` 通用组件可以保留，但音乐库和歌曲详情不再使用它。
- 搜索感应区提升到 TopBar 之上、Dialog 之下，恢复顶部中央 26px 移入唤醒；感应区与搜索框设置 Electron `no-drag`，TopBar 其余区域继续可拖动。

## 三、滚动与禁止误选文字

- 所有 renderer 自有滚动容器统一添加 `data-scroll-region`，隐藏滚动条但保留滚轮、触摸板、触摸和键盘滚动，并设置 `overscroll-behavior: contain`。
- 不新增服务端分页；搜索等有限结果集使用无滚动条的正常滚动，长歌曲列表继续使用现有 DOM 虚拟窗口算法。
- 应用壳、按钮、卡片、歌词、侧栏和舞台默认使用 `user-select: none` / `-webkit-user-select: none`，避免滑动或拖动时选中标题、歌名和歌词。
- `input`、`textarea`、`[contenteditable]` 及明确标记为 `data-allow-text-selection` 的日志/可复制区域恢复文本选择，不能为了禁止误选而破坏搜索框编辑、快捷键和复制能力。
- 统一改用 Pointer Events：识别到有效拖动后调用 `setPointerCapture()`，拖动期间挂载单一 `data-pointer-dragging` 状态，结束、窗口失焦、页面隐藏和组件卸载时都必须释放。
- 拖动阈值统一为约 3px；阈值前仍视为点击，超过阈值后才消费事件，避免按钮轻微移动后失效。
- 对封面、壁纸预览等非文件拖放目标设置 `draggable=false`，并阻止对应 `dragstart`，不出现浏览器图片幽灵和禁止光标。
- 只在舞台确实消费拖动/滚轮时调用 `preventDefault()`；UI 滚动区、输入控件和可选择文本区域必须交还浏览器原生行为。

## 四、壁纸层与歌词层

### 4.1 分层模型

- **壁纸层（Background/Wallpaper Layer，底层）**：在任一时刻承载一种背景来源：
  - Three.js 音乐视觉；
  - 本地图片；
  - 本地视频；
  - 可导入的 Wallpaper Engine 媒体项目。
- **歌词层（Lyrics Layer，上层）**：承载 `Lyrics3DLayer`，其可见性、旋转、景深、拖动和缓动不再依赖 `backgroundEnabled`，在上述任意壁纸来源下都可以工作。
- **UI 层**位于两者之上，但不属于沉浸画面的两层；TopBar、搜索、侧栏、设置和播放器按各自命中区域优先接管输入。
- Three.js 背景对象和 3D 歌词可继续位于同一 scene 中，但必须拆成独立 group、独立目标变换和独立更新分支；不得再通过复制背景粒子 rotation 的方式隐式控制歌词。
- `backgroundEnabled` 只控制粒子、Bloom、封面纹理、背景转场和相关音频计算；歌词层更新与歌词交互始终运行。

### 4.2 歌词交互

- 指针命中歌词交互区域时，歌词层在 Three.js 视觉、图片、视频和 Wallpaper Engine 背景下都拥有相同行为：
  - 左键按住并拖动：对歌词组做 3D yaw/pitch 旋转，使用 Pointer Capture，释放后结束拖动；这就是“歌词可以点击拖放”的交互。
  - 滚轮/触摸板：调整歌词层的 3D 景深或缩放目标，使用缓动并设上下限，不直接修改壁纸相机。
  - 拖动过程中禁止文本选择，不触发页面原生 HTML drag-and-drop。
- 歌词命中区域只覆盖实际歌词的可交互投影范围或一个明确的歌词交互容器，不能用全屏透明层阻断侧栏、搜索和空白背景。
- 歌词的 rotation/depth 目标单独保存；切歌只更新歌词内容，不重置用户当前观察角度。首次启动使用旧版默认角度，本轮不新增复杂的手势配置项。
- 规范化 `WheelEvent.deltaMode` 并限制单次增量，兼容普通滚轮和高精度触摸板，避免突跳。

### 4.3 壁纸交互与事件优先级

事件按以下顺序路由，命中后不再继续传给下一层：

| 优先级 | 指针位置/背景类型 | 滚轮 | 点击拖动 |
| --- | --- | --- | --- |
| 1 | 搜索、歌单、设置、日志等 `data-scroll-region` 或原生控件 | 原生滚动/控件行为，不改变舞台 | 原生控件行为 |
| 2 | 歌词交互区域，任意壁纸类型 | 只调整歌词 3D 景深/缩放 | 只旋转歌词 3D 组 |
| 3 | 空白背景，Three.js 3D 音乐视觉 | 调整背景相机/封面景深 | 保留 Three 背景自身允许的封面旋转 |
| 4 | 空白背景，图片、视频或 Wallpaper Engine 媒体 | 不响应，不 `preventDefault()` | 不旋转、不拖动壁纸 |

由此保证：歌词始终可操作；普通壁纸永远不抢滚轮；只有当前壁纸来源确实是 Three.js 3D 视觉时，空白背景上的滚轮才操作壁纸层；同一次滚轮不得同时改变歌词和壁纸。

## 五、Three.js 升级与新增 3D 背景

### 5.1 升级策略

- 先记录当前六套视觉的截图、帧率、显存释放、窗口缩放和设备丢失恢复基线，再将 `three`、`@types/three` 从 `0.128.0` 同步升级到锁定目标版本。
- 单独处理新版颜色空间、纹理编码、渲染目标、后处理、ShaderMaterial、BufferGeometry 和废弃 API 差异；不得用 `any` 或关闭类型检查掩盖迁移问题。
- 保留现有 shader 数学与视觉参数，只有新版 API 不兼容时才做最小修改；每个旧预设逐一截图对比，不能把“能编译”当作迁移完成。
- 继续使用项目全局 Ticker。Three 渲染、GSAP 驱动的 Three 对象属性和歌词更新不得各自创建裸 `requestAnimationFrame`；暂停、最小化和省电状态仍由统一性能状态机控制。

### 5.2 新增背景

在现有六套预设之后追加至少三套，不复用或改变旧数字 ID：

1. **星云隧道**：实例化粒子与雾化纵深形成可随节拍推进的空间隧道。
2. **晶体波场**：低面数晶体/网格根据频段产生波浪、折射色与高光变化。
3. **几何天际线**：实例化几何柱体形成环形城市，低频控制高度、节拍触发光带传播。

每套背景实现统一的 `init/update/resize/dispose` 生命周期、质量分级和降级路径；限制 draw call、像素比和后处理开销。滚轮只修改当前 Three 背景相机/景深，歌词仍使用独立变换。

## 六、玻璃组件与动画基础设施

- 安装并锁定 `react-glass-ui`、`gsap`，新增本地 `GlassSurface` 适配组件统一主题 token、圆角、折射强度、禁用态、键盘焦点及低性能降级。
- 优先替换重复手写的普通面板、弹层、按钮外壳和浮层玻璃 CSS；经典主题专用核心 SVG/filter 先保留，通过适配层复用，视觉对比通过后再决定是否移除旧实现。
- GSAP 替换手写的定时器/CSS class 编排动画：侧栏进出、搜索框唤醒、弹层展开、音质菜单以及歌词入场/退场。业务状态仍由 React/zustand 管理，GSAP 只负责插值与时间轴。
- React 中使用 `gsap.context()` 并在 effect 清理时 `revert()`/kill timeline；快速切歌、重复展开、卸载和窗口失焦时不得遗留 tween。
- 遵守 `prefers-reduced-motion` 和性能档位：减少动态时缩短或取消位移/旋转，低性能模式关闭昂贵折射与非必要粒子动画。

## 七、逐字歌词与歌词动画

### 7.1 数据模型

- 将歌词渲染模型扩展为“行包含字/词片段”：每个片段具有文本、开始时间、持续时间；优先使用网易云 YRC 等逐字时间戳，保留原始行时间与翻译。
- “一个字一个字”是渲染与时间轴粒度，不把 Unicode 字符简单按 UTF-16 拆分；使用 grapheme segmentation，避免拆坏 emoji、组合字符和特殊标点。
- 对只有普通 LRC 行时间、没有逐字时间的数据，按可见字符权重在本行到下一行之间生成估算时间；明确标记为 `estimated`，避免伪装成精确逐字歌词。时间不足或纯音乐时回退整行显示。
- 保持解析、时间估算与动画表现分离；共享歌词模型可序列化，不保存 GSAP 实例或 Three 对象。

### 7.2 动效

至少提供三种歌词动效并允许在“音乐视觉”设置中选择，默认使用性能稳定的逐字高亮：

1. **逐字流光**：播放进度逐字填充颜色、亮度和轻微缩放。
2. **弹性浮现**：字词按时间从景深后方淡入并轻微回弹。
3. **波浪翻转**：当前字附近形成受限的纵向波浪/3D 翻转，远离播放点后复位。

- 动画时间轴以音频 `currentTime` 为真值，seek、暂停、倍速、缓冲和切歌时直接校准进度，不能依赖 tween 自己累计时间而漂移。
- 只为当前行及前后少量行创建活动动画；离屏歌词不保留 tween。歌词拖动/滚轮操作期间动画继续，但用户变换与字词动画写入不同的 group/属性，避免互相覆盖。

## 八、播放音质控件重做

- 播放器外部常驻位置直接显示**实际播放音质**，例如“标准”“极高”“无损”“Hi-Res”；尚未解析时显示用户期望音质并标注“待确认”，不能只显示笼统的“音质”。
- 点击当前音质标签即切换/展开音质选项；菜单始终向上展开（播放器位于底部时使用 top/side placement），不得因自动碰撞翻到下方。空间不足时向上限高并在菜单内部滚动。
- 不沿用当前原生 `select + label + em` 的控件风格。改为紧凑的无边框文字/徽标触发器和独立上浮菜单，选中项、不可用项、实际降档和试听限制需要清晰但不过度装饰。
- 选择后立即更新偏好并重新解析当前歌曲；外部标签最终以服务端返回的 `resolvedQuality` 为准。请求期间防重复提交，失败时恢复实际档位并给出非阻塞提示。
- 支持键盘打开、上下选择、Enter 确认、Escape 关闭、焦点返回与屏幕阅读器语义；菜单动画使用 GSAP，但定位和可访问性不能依赖动画完成。

## 九、实施顺序与主要文件范围

1. 先收敛经典主题及旧持久化迁移：`theme/types.ts`、`presets.ts`、`store.ts`、`persistence.ts`、设置 UI 与相关测试。
2. 再统一滚动区标记、文本选择和 Pointer Events：`App.tsx`、shell 组件、`global.css`、`m3.css`。
3. 升级 Three.js 并逐个迁移六套旧预设，通过视觉基线后再追加三套新背景：`package.json`、lockfile、`visual/stage.ts`、`visual/shaders.ts`、`visual/presets/`。
4. 引入 `react-glass-ui`/GSAP 适配层，再分批替换普通玻璃外壳与手写动画；不一次性删除经典核心资产。
5. 拆分歌词/壁纸变换和输入路由，并扩展逐字歌词模型与三套歌词动效：`shared/lyrics/`、`features/lyrics/`、`visual/lyrics3d/`。
6. 最后重做播放器音质触发器和始终向上的菜单：`App.tsx`、播放器样式、`playback/quality.ts` 及交互测试。

## 十、验收标准

### 10.1 界面与基础交互

- 设置中不再出现主题预设选择或主题自定义面板；首次安装、旧主题数据升级和重置后都只使用经典主题。
- TopBar 完全透明且窗口拖拽正常；左右侧栏可移入展开、2 秒延迟关闭并重新移入取消关闭。
- 搜索栏可从顶部中央唤醒、输入、切换平台、滚动结果并选择歌曲。
- 所有应用内滚动条不可见，但搜索、歌单、设置、日志和壁纸列表仍可用滚轮、触摸板和键盘滚动。
- 拖动歌词、封面、侧栏或滑块时不会选中页面文字，也不会出现图片幽灵；输入框和明确可复制区域仍能选择文字。

### 10.2 双层舞台

- Three.js 音乐视觉下：歌词区域滚轮只改变歌词，空白背景滚轮只改变 Three 背景；两者不会同时变化。
- 图片、视频和 Wallpaper Engine 背景下：歌词仍可左键拖动旋转并使用滚轮调整 3D 景深；空白背景滚轮不改变壁纸或背景相机。
- 在搜索、歌单、设置等 UI 列表上滚动永远不会改变歌词或壁纸。
- 切换背景来源、切歌、窗口失焦或最小化后没有卡死的拖动状态；最小化仍遵守全局 Ticker 的零视觉开销策略。
- 歌词与背景变换完全解耦，`backgroundEnabled=false` 不再暂停歌词层更新。

### 10.3 视觉、歌词与音质

- Three.js 与类型包升级到同一锁定版本，原六套预设无明显视觉回退，新三套背景可选择、可释放且在性能降级时稳定。
- `react-glass-ui` 与 GSAP 只通过适配/生命周期规范使用；反复开关侧栏、搜索、菜单和切歌后无残留动画、计时器或 GPU 资源。
- 有逐字时间戳的歌曲逐字同步准确；普通 LRC 使用估算时间且可稳定 seek，三种歌词动效均不影响歌词 3D 拖动和滚轮。
- 音质外部标签始终显示具体实际音质；点击后菜单只向上展开，键盘可操作，切换成功、降档和失败状态均正确反馈。

## 默认约定

- “只保留经典”仅针对界面主题；六个现有 Three.js 音乐视觉预设继续存在，新背景只在其后追加。
- “双层”是壁纸职责层与歌词职责层，不等于必须创建两个 Three.js scene、两个 canvas 或两个 Ticker。
- “歌词点击拖放”定义为在歌词命中区域按住左键拖动 3D 旋转，抬起结束；不实现文件拖放，也不默认改变歌词 DOM 布局位置。
- 普通图片、视频和 Wallpaper Engine 媒体壁纸没有滚轮交互；歌词覆盖其上时仍可独立交互。
